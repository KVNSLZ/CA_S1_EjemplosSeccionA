# Buenas prácticas al usar Markdown

Antes de comenzar, es importante seguir algunas **buenas prácticas para documentación en Markdown**:

## Usar títulos jerárquicos

Markdown utiliza `#` para crear encabezados.

Ejemplo:

```
# Título principal
## Sección
### Subsección
#### Detalle
```

Esto permite que el documento sea **más fácil de leer y navegar en GitHub**.

---

## Mantener una estructura clara

Un buen documento técnico debe seguir un flujo lógico:

1. Introducción
2. Objetivos
3. Descripción del tema
4. Planificación
5. Desarrollo
6. Resultados
7. Reflexión y conclusiones

---

## Usar tablas para información estructurada

Las tablas permiten organizar mejor la información.

Ejemplo:

| Elemento  | Descripción                 |
|-----------|-----------------------------|
| Tema      | Tema de la capacitación     |
| Público   | Audiencia objetivo          |
| Duración  | Tiempo estimado             |

---

## Usar listas para organizar ideas

Las listas ayudan a explicar conceptos de forma clara.

Ejemplo:

- Punto importante 1
- Punto importante 2
- Punto importante 3

---

## Usar bloques de código cuando sea necesario

Ejemplo:

```
git clone https://github.com/usuario/repositorio
```

Nota: Se puede hacer con muchos lenguajes de programación, y con casi cualquier implementación siempre y cuando al inicio se le agregue el nombre de a quien hace referencia, python, cpp, java, etc.
